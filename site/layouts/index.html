How it works

Three steps to a multi-transport gateway.

1

Plug in devices

Connect your Meshtastic radio, satellite modem, or cellular module via USB. MeshSat auto-detects hardware via VID:PID and protocol probing.

2

Define routing rules

Set access policies, failover groups, and transform pipelines from the web dashboard. Control which messages go where, at what cost.

3

Messages flow

The dispatcher routes messages across transports with failover, compression, encryption, and delivery tracking. No cloud needed.

Built for the field

Everything you need to maintain communications when infrastructure fails.

Policy-Driven Routing

Access rules engine with object groups, rate limiting, failover resolution, and implicit deny.

Transform Pipeline

Per-interface egress/ingress transforms: zstd, SMAZ2, MSVQ-SC compression + AES-256-GCM encryption.

USB Hot-Swap

DeviceSupervisor with VID:PID identification cascade, automatic protocol probing, and claim-based port management.

Satellite Pass Prediction

SGP4/TLE propagation with 4-mode state machine (Idle, PreWake, Active, PostPass) for power-efficient operation.

3-Tier Compression

SMAZ2 (lossless, <1ms), llama-zip (LLM-based, ~200ms), MSVQ-SC (lossy semantic, rate-adaptive).

Field Intelligence

Dead man's switch, polygon geofences, channel health scoring, satellite burst queue, and mesh topology visualization.
